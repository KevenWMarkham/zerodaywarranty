Executive summary · the one paragraph

The Zero Day Warranty solution is a set of capabilities that compose end-to-end. This document lays them out as swim lanes: the rows are the capability owners (consumption, data plane, agent orchestration, accelerated analytics, human-in-the-loop, downstream action) over an always-on governance foundation; the columns are the seven phases of an investigation, from cluster detection to action. Each cell is a discrete capability. Two more lanes cover the Day-0 prevention path (catch the defect at the build station) and a full capability catalog that enumerates all 30+ capabilities with their platform component, agent step, and persona owner.

01How to read these lanes

A swim lane diagram answers two questions at once: what the capability is, and who owns it. Rows are owners; columns are the phases of one warranty root-cause investigation. The same lanes run for every cluster the platform investigates.

02Master capability swim lane

Six owner lanes across seven phases, over the always-on governance foundation. Every box is a capability the solution ships. Steps in [n] reference the 24-step agent chain.

03Day-0 prevention swim lane

The second reading of "Zero Day Warranty" — catch the defect at the build station so it never becomes a claim. A separate, station-local capability set running on NVIDIA Metropolis, feeding the same medallion and audit foundation.

Two readings, one operating model

The agent chain (Figure 1) gives zero-day investigation latency; the build-station layer (Figure 2) gives Day-0 detection. Both write to the same Fabric medallion, run under the same Entra identity, and seal to the same Purview audit substrate. Optional, but together they are the full story.
